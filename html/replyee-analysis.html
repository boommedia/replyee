What to Actually Build

Replyee — A Chatbase Lite for Boom Media Clients

Following the Boom Media naming pattern. A white-labeled AI chatbot platform for your clients — trained on their content, embedded on their websites, managed by Boom Media.

MVP Feature Set (What to Build)

✓
Embeddable chat widget

Small JS snippet that drops onto any client website. Pops a floating chat bubble.

✓
Knowledge base ingestion

Upload PDFs, paste URLs, add text — chunked and stored as vectors in Supabase pgvector.

✓
AI responses via Claude

RAG pipeline: user message → vector search → context → Claude → answer. Fast, accurate, on-brand.

✓
Client dashboard

View conversation history, train the bot, customise colours/logo, get embed code.

✓
Email handoff (lead capture)

When bot can't answer, collect visitor email + question. Send to client via Resend.

✓
Basic analytics

Conversation count, popular questions, unresolved rate. Simple charts, no complexity.

✓
Boom Media admin panel

You manage all client bots from one place. Add clients, update knowledge bases, monitor usage.

What to Skip (Phase 2 or Never)

✗
Live human chat

Adds massive complexity (websockets, agent queues, SLAs). Use email handoff instead.

✗
WhatsApp / Instagram channels

API approvals, webhook management, paid add-ons. Phase 2 if demand exists.

✗
Video / audio calling

Crisp feature. Overkill for an AI chatbot product. Skip entirely.

✗
Ticketing system

Intercom territory. Email handoff covers 90% of the same need at 10% of the build cost.

✗
300+ integrations

Zapier webhook is enough. Let clients connect whatever they want via webhook.

✗
Self-serve billing (phase 2)

Start by billing clients manually via Boom Media invoices. Add Stripe later when you have 10+ clients.

Chunking, embedding, vector search, context window management — not complex, but new concepts to implement well.

If a client's chatbot breaks at 2am, it's your problem. Chatbase handles that for you now.

💸

📉
Chatbase keeps improving

They have a big team iterating fast. Your lite version may fall behind on features unless you invest in it.

🎛️
Content moderation is your problem

If a client's bot says something wrong or harmful, you're liable. Need guardrails + system prompts.

Build Timeline

8–12 Week MVP Plan

Solo founder with Claude Code. Follows your proven Next.js / Supabase pattern.

1

Foundation — RAG Pipeline + Database Schema

Weeks 1–2

Enable pgvector in Supabase. Build document ingestion (PDF, URL, text → chunks → embeddings → stored). Wire Claude Haiku for response generation. Test end-to-end with a dummy knowledge base. No UI yet.

2

Chat Widget — Embeddable JS Script

Weeks 3–5

The hardest part. Vanilla JS (no React) floating bubble widget. Loads async, no conflicts with client site CSS. API calls to your Vercel backend. Typing indicators, message history (session). Test on multiple site stacks (WordPress, Webflow, Squarespace, custom).

3

Client Dashboard + Knowledge Base UI

Weeks 6–8

Next.js dashboard for clients. Upload files, paste URLs, add text snippets. Preview the chatbot. Customise colours, name, avatar, greeting message. Get embed snippet. View recent conversations.

4

Admin Panel + Email Handoff + Analytics

Weeks 9–10

Boom Media admin view: all clients, all bots, usage stats. Email handoff when bot confidence is low (Resend). Basic analytics dashboard: conversations per day, top questions, unresolved rate.

5

Testing, Hardening + First Client Launch

Weeks 11–12

Rate limiting, usage caps per plan, prompt injection guards, content filtering. Onboard first 2–3 Boom Media clients. Gather feedback. Iterate.

Day 85: First paying client chatbot is live. Your existing Boom Media clients become the beta users — no cold start problem.
